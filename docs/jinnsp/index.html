FEATURES

Built for large
music libraries.

JinnSP focuses on practical playback and library control without locking your music into a platform.

Large library control

Manage large track sets, search the library, and filter by source and tags.

Playlists

Create playlists, play them directly, shuffle within the playlist, and search inside large lists.

Local and URL sources

Use local files where the browser or Android file access allows it, and play direct HTTP/HTTPS media URLs such as R2-hosted audio.

Playback tools

Queue, repeat, favorites, background audio, and Media Session metadata for supported lock screens and browsers.

Portable data

CSV import/export for editable lists and JSON-based backup/restore for app data.

Installable PWA

Runs from the browser and can be installed on supported desktop, Android, and iOS environments.

PLATFORMS

Clear about what
the web can do.

Capabilities vary by browser and operating system, especially for local files and lock screen controls.

Desktop

Direct URL playback, CSV workflows, backup/restore, and installable PWA behavior are supported by modern Chromium browsers. Local file persistence depends on browser file permissions.

Android

The Capacitor build keeps Storage Access Framework folder access, saved preferences, background playback, and Media Session support.

iOS

Safari and Home Screen PWA playback are supported within iOS limits. Lock screen next/previous buttons may depend on iOS Media Session behavior.

Offline

The app shell is cached by the service worker. Remote music URLs still require network access, and local file access depends on the platform permission model.
