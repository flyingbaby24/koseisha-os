│  ## dao_connection.md
│
│  ### 概要
│  本ファイルは、思想OSをDAO（分散型自律組織）に接続する際の思想的条件・構造的意味・参加型設計を示す。思想は単なるガバナンスルールではなく、問いと接続の構造に基づく自律的参加モデルである。
│
│  ### 接続条件と構造的役割
│
│  - **接続条件**：
│    - 思想理解度（values.mdに基づく思想接続スコア）
│    - 起動行為（flowchart.md参照：再起動プロセスの履歴）
│    - 可視化された参加記録（質問、提案、修正案など）
│
│  - **思想的役割**：
│    - 思想DAOは再起動者の流通レイヤーであり、資本DAOとは異なる
│    - 投票は「問い」によって起動され、「接続」によって重みが与えられる
│
│  ### 擬似スマートコントラクト例（思想スコア連動BI）
│  ```solidity
│  function computeBI(address user) public view returns (uint256) {
│      uint256 thoughtScore = Thought.getScore(user);
│      uint256 baseBI = 1000;
│      if (thoughtScore >= 80) return baseBI * 0.3;
│      if (thoughtScore >= 50) return baseBI * 0.6;
│      return baseBI;
│  }
│  ```
│  ※あくまで思想概念の反映例であり、実装は思想の分岐構造と一致する必要がある。
│
│  ### 思想接続 vs 資本接続
│
│  | 接続タイプ | 起動条件         | 分配の意味            | 破綻条件           |
│  |--------------|------------------|------------------------|--------------------|
│  | 資本DAO       | トークン保持       | 出資に対する見返り      | ガバナンス集中・格差再生 |
│  | 思想DAO       | 構造理解 + 共鳴履歴 | 再起動に対する共振保障 | 構造誤解・接続喪失     |
│
│  ### 再接続と流動性
│
│  DAO接続は恒久的ではない。思想理解や接続履歴は常に更新可能であり、過去の非接続状態は再起動によって上書き可能である。
│
│  参加者は自由に思想層と従属層を行き来し、構造を理解し再接続する権利を保有する。
│
│  > 「DAOとは、設計図のない組織ではなく、“問いの起動条件を内包した構造体”である。」
